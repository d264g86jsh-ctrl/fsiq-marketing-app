# Campaign Brief Template — Production SOP
**Version:** 1.1 | **Last updated:** May 2026
**Skill:** `skills/paid-media/campaign-brief-generator.skill.ts`
**Authoritative reference:** FSIQ-VIDEO-AD-33 (Exclusivity Video Ad)

---

## Purpose

This SOP defines the exact structure and content requirements for every FSIQ Campaign Production Brief.
The brief is generated as a `.docx` file and uploaded to SharePoint under:
`[Concept Folder]/Campaign Brief/[AD-ID]-Brief.docx`

It is the editor's single source of truth for recording, cutting, and delivering each video ad.

---

## Document Formatting Rules

- All text is **bold** throughout the entire document
- All tables use **borderless** styling (no visible cell borders)
- The entire document is structured as a series of borderless tables and paragraphs
- Section labels are bold table rows or bold standalone paragraphs
- No markdown syntax, no bracket tags visible in the output
- No "Full Script" dump section — script is structured per hook/body/CTA

---

## Section 1 — Header (Borderless Table, 2 Rows)

Single-column rows spanning full width:

**Row 1:** `FOODSERVICEIQ  |  Campaign Production Brief`
**Row 2:** `[AD-ID]  |  [Concept Name]  |  [Format]  |  WEEK-[##]`

Format = Hook type label (e.g., `Podcast Studio`, `iPhone`, `Both`)
Week = derived from current week number (`WEEK-32`, etc.)

---

## Section 2 — IDENTIFICATION (Borderless Table)

**Label row (full width):** `IDENTIFICATION`

Then 2-column rows:

| Field | Value |
|-------|-------|
| Ad ID | `FSIQ-VIDEO-AD-XX` |
| Ad Name | Concept name from creative_pipeline |
| Format | Hook type (e.g., `Podcast Studio`, `iPhone`, `Both`) |
| Week | `WEEK-##` |
| # of Hooks | `N  (Hook 1, Hook 2, …)` |
| # of Bodies | `1  (shared across all hooks)` |
| # of CTAs | `1  (shared across all hooks)` |
| Total Variations | `N videos  (N Hook + Body + CTA full cuts)` |
| Raw Footage & Audio | Concept name or footage folder name |

Counting rules:
- Count one hook per `[HOOK-IPHONE]`, `[HOOK-STUDIO]`, `[HOOK]`, `[HOOK 2]`, `[HOOK-2]`, etc.
- If both `[HOOK-IPHONE]` and `[HOOK-STUDIO]` exist: 2 hooks

---

## Section 3 — CAMPAIGN OBJECTIVE

**Label row (full width):** `CAMPAIGN OBJECTIVE`

One paragraph (2–3 sentences) generated by Claude using:
- Hook type, awareness level, lp_code from creative_pipeline
- First 2 sentences of the hook text

Describes:
1. What this ad is designed to make the viewer feel or do
2. Who it targets (revenue threshold, restaurant type)
3. What action it drives (download, book call, click link)
4. Funnel stage context (Unaware / Pain-Aware / Solution-Aware)

No bracket tags, no bullet points, no AI vocabulary. Plain operator language.

---

## Section 4 — TECHNICAL SPECS (Borderless Table)

**Label row (full width):** `TECHNICAL SPECS`

Then 2-column rows:

| Field | Value |
|-------|-------|
| Platform | `Meta (Facebook / Instagram)` |
| Aspect Ratios | `9:16 (primary)   \|   1:1 (secondary)` |
| Length | Derived from word count: ≤75w = ~30s, 76–120w = ~45s, 121–180w = ~60s, 181+ = ~90s |
| Style | Hook type label |
| Captions | `Required — sound-off friendly. Use FSIQ brand color palette: foodserviceiq.com` |
| Audio | `Original recorded audio only — no voiceover replacement` |
| Delivery | Tone description appropriate to hook type (see below) |

**Delivery by hook type:**
- Podcast Studio → `Conversational, measured, authoritative. Direct address — not salesy.`
- iPhone / POV → `Direct, candid, personal. Natural delivery — talking to one person.`
- Both → `Conversational throughout. iPhone hooks: candid and direct. Studio body: authoritative.`

---

## Section 5 — SCRIPT

**Label row (full width):** `SCRIPT`

For each hook, output a label paragraph then hook text paragraph. The awareness level is appended **inline** at the end of the hook text paragraph (not a separate line):

```
HOOK 1  —  Podcast Studio
[full hook text]  [UNAWARE]
```

The `[AWARENESS LEVEL]` tag at the end of the hook text is the **only** bracket-style label that appears in the output. It identifies the funnel stage for that hook. Uppercase, in brackets.

After all hooks:

```
BODY  —  Shared          ← label row (in table)
[body paragraph 1]
[body paragraph 2]
...
CALL TO ACTION  —  Shared  ← label row (in table)
[CTA paragraph 1]
[CTA paragraph 2]
```

**Parsing rules (critical — no bracket tags visible in output):**
- Strip: `[HOOK-IPHONE]`, `[HOOK-STUDIO]`, `[HOOK]`, `[HOOK 2]`, `[BODY]`, `[MIDDLE]`, `[CTA]`, `[CALL TO ACTION]`
- Preserve all other text verbatim, including ellipses, dollar signs, percentages
- Each line break in source = separate `Paragraph` node in the docx

---

## Section 6 — VIDEO STRUCTURE | Editor Reference

**Label paragraph:** `VIDEO STRUCTURE  |  Editor Reference`

For each hook:
```
HOOK [N]  —  [Format]
[full hook text — clean]
Editing Direction: Seamless cut into body
```

For body:
```
BODY  —  Shared across all [N] variations
Editing Direction: [direction — e.g., Camera shut transition – side angle]
[body paragraph 1 with inline camera notes if applicable]
Editing Direction: Seamless transition
[body paragraph 2]
...
Editing Direction: Seamless transition into CTA
```

For CTA:
```
CTA  —  Shared across all [N] variations
[CTA paragraph 1]
[CTA paragraph 2]
Editing Direction: Seamless close
```

Default editing directions by hook type:
- Podcast Studio: `Seamless cut into body`
- iPhone: `Direct cut into body`
- Body opening: `Camera shut transition – side angle` (if studio); `Direct cut` (if iPhone)

---

## Section 7 — EDITOR NOTES (Borderless 2-Column Table)

**Label row (full width):** `EDITOR NOTES`

Then 2-column rows where left = sub-label, right = content paragraphs:

| Sub-label | Content |
|-----------|---------|
| `AUDIO` | Studio audio quality requirements for the format |
| `CAPTIONS — IMPORTANT NOTE` | Specific caption formatting rules (contractions, line length, etc.) |
| `FINAL NOTES` | Multi-line: `Tone: …` / `Pacing: …` / `Clarity: …` / `Framing: …` / `Transitions: …` / `Captions: …` |

**FINAL NOTES format (each on its own line):**
```
Tone: [tone description]
Pacing: [pacing description]
Clarity: Ad must be fully understandable with sound OFF. Captions are required.
Framing: [framing description — e.g., Neil centered and occupying ~90% of the shot]
Transitions: All cuts between hook, body, and CTA should be seamless.
Captions: Format consistent with FSIQ brand and previous videos.
```

**Per hook type defaults:**
- Podcast Studio: `Podcast studio audio only. Zero background noise. Ensure clean, broadcast-quality audio throughout — no echo, no compression artifacts.`
- iPhone: `Record in quiet environment. No background music. Natural room tone acceptable.`

---

## Section 8 — CAPTIONS

**Label paragraph:** `CAPTIONS`

Then for each hook, one label paragraph followed by the clean caption text:
```
HOOK 1
[clean hook text — no brackets, ellipses cleaned for readability]

HOOK 2
[clean hook text]

...

BODY
[clean body paragraph 1]
[clean body paragraph 2]

CTA
[clean CTA paragraph 1]
[clean CTA paragraph 2]
```

This is **not a table** — it's a series of paragraphs. Labels are bold uppercase, text follows on the next paragraph.

Cleaning rules for captions:
- Strip all `[BRACKET]` tags
- Preserve `...` ellipses as `…`
- Preserve dollar signs, percentages, numbers exactly
- Do not paraphrase — match speech exactly
- One paragraph per spoken sentence/breath

## Full Document Section Order

1. Header table (2-row, full-width, borderless)
2. IDENTIFICATION label row + data rows (borderless table)
3. CAMPAIGN OBJECTIVE label row + paragraph
4. TECHNICAL SPECS label row + data rows (borderless table)
5. SCRIPT label row + hook paragraphs + BODY label row + body paragraphs + CTA label row + CTA paragraphs
6. VIDEO STRUCTURE paragraph + editing direction paragraphs
7. EDITOR NOTES label row + AUDIO / CAPTIONS / FINAL NOTES rows (borderless table)
8. CAPTIONS paragraph + label/text paragraph pairs

## Script Section Detection

| Bracket tag | Maps to |
|-------------|---------|
| `[HOOK-IPHONE]` | Hook 1 (iPhone format) |
| `[HOOK-STUDIO]` | Hook 1 or 2 (Studio format) — if both IPHONE and STUDIO exist, STUDIO = Hook 2 |
| `[HOOK]` | Hook 1 (infer format from hook_type in pipeline) |
| `[HOOK 2]` or `[HOOK-2]` | Hook 2 |
| `[HOOK N]` or `[HOOK-N]` | Hook N |
| `[BODY]` or `[MIDDLE]` | Shared body |
| `[CTA]` or `[CALL TO ACTION]` | Shared CTA |

If no bracket tags: first 3 lines = hook, last 2 = CTA, middle = body.

## SharePoint Upload Location

```
Sales & Marketing/Marketing/Ad Campaigns/Ad Creatives/Video Creatives/
  └── [AD-ID] - [Concept Name]/
        └── Campaign Brief/
              └── [AD-ID]-Brief.docx
```

Concept folder item ID lookup (excludes subfolders):
```sql
display_name ILIKE '[AD-ID]%'
AND item_type = 'folder'
AND path ILIKE '%Video Creatives/[AD-ID]%'
AND path NOT ILIKE '%[AD-ID]%/%'
```
